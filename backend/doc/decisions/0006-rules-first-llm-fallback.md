# ADR-0006：LLM 规则优先、分级触发；Phase 1 零 LLM 上线

- **状态**：已采纳
- **日期**：2026-07-03（设计评审期拍板）
- **来源**：设计文档 4.1.3、4.5、第九章

## 背景

团队无专职运营且 LLM 成本敏感；表单与规范 Markdown 两个通道的内容本身已结构化，不需要 LLM 重组。

## 决策

重组流水线采用"规则优先、LLM 兜底"分级：规则能处理的不调 LLM；Phase 1 只开放表单 + 规范 Markdown 两通道，平台流水线侧零 LLM 上线。LLM 重组兜底随 P2 飞书接入交付（试点实测闭环成本后再放开）。

## 理由

MVP 闭环不依赖 LLM 即可成立；成本与质量风险后置到有真实数据支撑时再决策。

## 边界说明

"零 LLM"仅指平台流水线侧——OpenViking 写入后自动生成 L0/L1 摘要、SDK 不可跳过（已查证），模型网关是 P1 就存在的依赖，成本已计入设计 4.5。
