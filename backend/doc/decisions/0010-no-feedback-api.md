# ADR-0010：不做 /v1/feedback，质量发现内部化

- **状态**：已采纳
- **日期**：2026-07-04
- **来源**：设计文档 4.4.2（已按此改写）、6.3；第六章评审拍板

## 背景

早期方案考虑提供 /v1/feedback 让 Agent 回报"这条知识答错了"。但 Agent 侧反馈质量不可控（误报、无上下文），且需要额外的反馈治理流程。

## 决策

不做 /v1/feedback 接口。质量发现内部化：审计留存完整查询记录（query + 命中 kid/version）+ knowledge_version 版本快照 + 控制台人工打标，三者合起来还原并标记问题知识。**审计 = 治理数据主源**。

## 理由

kid + version + 快照已能精确还原"Agent 当时拿到了什么"，人工打标比 Agent 自动反馈信噪比高；少一个对外接口就少一套滥用防护。联动改动：4.4.4 增加"未审阅积压量"监控、4.2 增加版本快照、架构与生命周期画板中 feedback 引用已清理。
