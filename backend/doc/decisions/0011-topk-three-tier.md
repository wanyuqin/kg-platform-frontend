# ADR-0011：top_k 三层取值规则

- **状态**：已采纳
- **日期**：2026-07-04
- **来源**：设计文档 6.1.1；第六章评审拍板

## 背景

不同类型知识的合理召回数不同（FAQ 宜多、政策宜少），Agent 也可能有自己的偏好；需要一个确定的优先级规则。

## 决策

```text
生效值 = min( 入参 top_k ?? 类型级配置 ?? 平台默认 5, 硬上限 20 )
```

类型级配置存于 `domain.type_topk`（JSONB），仅当请求 type 恰为单一类型时启用，取 Key 白名单内各 domain 配置的最大值。

## 理由

入参 > 配置 > 默认的覆盖顺序符合直觉；硬上限 20 防止大 top_k 拖垮检索延迟与下游 token 消耗。
