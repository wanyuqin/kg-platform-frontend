# ADR-0020：expired 语义 P1 用查询时兜底过滤实现

- **状态**：已采纳
- **日期**：2026-07-04（MVP 技术设计阶段）
- **来源**：技术设计文档 四；设计文档 4.4.1

## 背景

"过期知识不被检索引用"是设计 4.4.1 的核心治理语义，但过期扫描任务排在 P3。

## 决策

P1 不建扫描进程，检索与取全文在查询时按 `status='published' AND expire_date >= 今天` 过滤，剔除者计入 excluded_expired。P3 扫描任务只负责状态落库 + 发复审卡片，不改变对外行为。

## 理由

对外语义 P1 首日即完整（不会出现"活动结束了规则还在被引用"），实现只是一个 WHERE 条件；状态落库与人工复审流程可以安全后置。
