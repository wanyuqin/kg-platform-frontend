# ADR-0012：平台自发 API Key：单 key、默认 10 QPS

- **状态**：已采纳
- **日期**：2026-07-04
- **来源**：设计文档 6.1.3；技术设计文档 十

## 背景

Agent 接入需要凭证体系。备选：对接公司统一凭证平台，或平台自发；以及是否支持双 key 轮换。

## 决策

平台自发 API Key（明文 `kp_{key_id}_{secret}`，签发响应中仅出现一次，库内只存 SHA-256）；默认限额 10 QPS、按 Agent 可调；**不做双 key 轮换**，泄露走吊销重发。

## 理由

MVP 阶段接入方数量有限，自发体系实现成本低、无外部依赖；双 key 轮换的运维收益在当前规模下不成立，吊销 + 重发（缓存主动失效、即时生效）已满足安全要求。
