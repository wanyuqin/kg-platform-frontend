# ADR-0004：知识按条存储：一条知识 = 一个文件

- **状态**：已采纳
- **日期**：2026-07-03（设计评审期拍板）
- **来源**：设计文档 2.2、4.1.4、5.2

## 背景

治理动作（过期、降权、下架、更新）的粒度是单条知识；OpenViking 的 add_resource 会对导入内容二次切分，破坏平台已拆好的条目边界。

## 决策

一条知识 = 一个 `viking://` 文件，用 `content/write(path, content)` 写入，路径与文件名全由平台指定；**禁用 add_resource**。

## 理由

独立过期 / 降权 / 下架的治理粒度与精确更新的前提；检索命中即完整条目，Agent 拿到的内容自包含。
