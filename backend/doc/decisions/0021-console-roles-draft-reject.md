# ADR-0021：控制台三级角色、草稿可见性、驳回回退

- **状态**：已采纳
- **日期**：2026-07-04（评审后补充决策，用户授权按最优判断定）
- **来源**：设计文档 7.1 / 7.2、4.2（状态机加 pending_review→draft 线）

## 背景

设计文档评审通过后，控制台交互细节需要收口才能开发。

## 决策

1. **角色三级**：平台管理员 / domain 管理员 / domain 成员（权限见 [console.md](../modules/console.md)）；
2. **草稿**：不是独立实体，是知识列表的 draft 筛选视图；仅本人可见；超 30 天未提交自动清理；
3. **驳回**（P2）：审核驳回必填理由，知识回退 draft 由提交人修改重提——状态机新增 pending_review→draft 转移线；
4. **人工待补齐队列**并入审核待办页三 tab（P2）；
5. **发布→可检索有秒级延迟**（index_state 索引中状态），控制台如实展示；
6. **拆分预览确认页**为 Markdown 上传与飞书首次导入共用，后续增量全自动。

## 理由

三级角色是权限模型的最小充分集；草稿视图化避免额外实体与页面；驳回回退闭环让"改完重提"不需要新建条目。
